Installation

⚙️ Installation

Follow these steps to install and run the template on your local machine:

Step 1: Extract Files

Extract the downloaded ZIP file to your desired location.

# Navigate to your projects directory
cd ~/projects

# Extract the ZIP file
unzip boost-web-agency.zip

# Navigate to the project directory
cd boost-web-agency

Step 2: Install Dependencies

Install all required npm packages:

# Using npm
npm install

# Or using yarn
yarn install

# Or using pnpm
pnpm install

⚠️ Note:

The installation may take 2-5 minutes depending on your internet speed. If you encounter any errors, try deleting the node_modules folder and package-lock.json file, then run the install command again.

Step 3: Environment Setup

Create a .env.local file in the root directory:

# Create environment file
touch .env.local

Add the following environment variables:

# Email Configuration (for contact form)
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-password
EMAIL_TO=recipient@example.com

# Optional: Database Configuration
DATABASE_URL=postgresql://user:password@localhost:5432/dbname

# Optional: Authentication
NEXTAUTH_SECRET=your-secret-key-here
NEXTAUTH_URL=http://localhost:3000

# Optional: Payment Gateway
STRIPE_PUBLIC_KEY=pk_test_...
STRIPE_SECRET_KEY=sk_test_...

Step 4: Run Development Server

Start the development server:

# Using npm
npm run dev

# Or using yarn
yarn dev

# Or using pnpm
pnpm dev

✅ Success!

Open your browser and navigate to http://localhost:3000 to see your application running.

Step 5: Build for Production

When ready to deploy, build the production version:

# Build the application
npm run build

# Start production server
npm run start

Troubleshooting

🔧 Troubleshooting

❌ Common Issues & Solutions

1. Module Not Found Error

Error: Module not found: Can't resolve 'xyz'

Solution:

  1. Delete node_modules folder
  2. Delete package-lock.json
  3. Run npm install again
  4. Restart development server

2. Port Already in Use

Error: Port 3000 is already in use

Solution:

# Use a different port`
`npm run dev -- -p 3001
`
`# Or kill the process using port 3000`
`# On Mac/Linux:`
`lsof -ti:3000 | xargs kill -9`
`
`# On Windows:`
`netstat -ano | findstr :3000`
`taskkill /PID [PID_NUMBER] /F

3. Images Not Loading

Error: Images show broken or don't load

Solution:

  • Ensure images are in the public folder
  • Image paths must start with / (e.g., /logo.png)
  • Check file names match exactly (case-sensitive)
  • Use Next.js Image component for optimization
  • Restart development server after adding new images

4. Environment Variables Not Working

Error: process.env.VARIABLE_NAME is undefined

Troubleshooting

🔧 Troubleshooting

❌ Common Issues & Solutions

1. Module Not Found Error

Error: Module not found: Can't resolve 'xyz'

Solution:

  1. Delete node_modules folder
  2. Delete package-lock.json
  3. Run npm install again
  4. Restart development server

2. Port Already in Use

Error: Port 3000 is already in use

Solution:

# Use a different port`
`npm run dev -- -p 3001
`
`# Or kill the process using port 3000`
`# On Mac/Linux:`
`lsof -ti:3000 | xargs kill -9`
`
`# On Windows:`
`netstat -ano | findstr :3000`
`taskkill /PID [PID_NUMBER] /F

3. Images Not Loading

Error: Images show broken or don't load

Solution:

  • Ensure images are in the public folder
  • Image paths must start with / (e.g., /logo.png)
  • Check file names match exactly (case-sensitive)
  • Use Next.js Image component for optimization
  • Restart development server after adding new images

4. Environment Variables Not Working

Error: process.env.VARIABLE_NAME is undefined

Solution:

  • Ensure .env.local file exists in root directory
  • Variable names must start with NEXT_PUBLIC_ for client-side access
  • Restart development server after changing env variables
  • Check for typos in variable names
  • Don't use quotes around values in .env file

5. Build Errors

Error: Build fails with TypeScript or ESLint errors

Solution:

# Check for TypeScript errors`
`npx tsc --noEmit
`
`# Run linter`
`npm run lint`
`
`# Fix auto-fixable issues`
`npm run lint -- --fix

6. Styling Not Applied

Error: Tailwind CSS classes not working

Solution:

  • Ensure globals.css is imported in app/layout.tsx
  • Check Tailwind configuration in tailwind.config.ts
  • Clear Next.js cache: rm -rf .next
  • Restart development server

7. Contact Form Not Sending Emails

Error: Form submits but no email received

Solution:

  • Check environment variables are set correctly
  • Verify Gmail app password (not regular password)
  • Check spam/junk folder
  • Look for errors in browser console and terminal
  • Test with a different email provider

8. Slow Performance

Issue: Application loads slowly

Solution:

  • Optimize images (use WebP format, compress files)
  • Use Next.js Image component for automatic optimization
  • Implement lazy loading for images and components
  • Minimize JavaScript bundle size
  • Use production build for testing: npm run build && npm start

🔧 Troubleshooting

❌ Common Issues & Solutions

1. Module Not Found Error

Error: Module not found: Can't resolve 'xyz'

Solution:

  1. Delete node_modules folder
  2. Delete package-lock.json
  3. Run npm install again
  4. Restart development server

2. Port Already in Use

Error: Port 3000 is already in use

Solution:

# Use a different port`
`npm run dev -- -p 3001
`
`# Or kill the process using port 3000`
`# On Mac/Linux:`
`lsof -ti:3000 | xargs kill -9`
`
`# On Windows:`
`netstat -ano | findstr :3000`
`taskkill /PID [PID_NUMBER] /F

3. Images Not Loading

Error: Images show broken or don't load

Solution:

  • Ensure images are in the public folder
  • Image paths must start with / (e.g., /logo.png)
  • Check file names match exactly (case-sensitive)
  • Use Next.js Image component for optimization
  • Restart development server after adding new images

4. Environment Variables Not Working

Error: process.env.VARIABLE_NAME is undefined

Solution:

  • Ensure .env.local file exists in root directory
  • Variable names must start with NEXT_PUBLIC_ for client-side access
  • Restart development server after changing env variables
  • Check for typos in variable names
  • Don't use quotes around values in .env file

5. Build Errors

Error: Build fails with TypeScript or ESLint errors

Solution:

# Check for TypeScript errors`
`npx tsc --noEmit
`
`# Run linter`
`npm run lint`
`
`# Fix auto-fixable issues`
`npm run lint -- --fix

6. Styling Not Applied

Error: Tailwind CSS classes not working

Solution:

  • Ensure globals.css is imported in app/layout.tsx
  • Check Tailwind configuration in tailwind.config.ts
  • Clear Next.js cache: rm -rf .next
  • Restart development server

7. Contact Form Not Sending Emails

Error: Form submits but no email received

Solution:

  • Check environment variables are set correctly
  • Verify Gmail app password (not regular password)
  • Check spam/junk folder
  • Look for errors in browser console and terminal
  • Test with a different email provider

8. Slow Performance

Issue: Application loads slowly

Solution:

  • Optimize images (use WebP format, compress files)
  • Use Next.js Image component for automatic optimization
  • Implement lazy loading for images and components
  • Minimize JavaScript bundle size
  • Use production build for testing: npm run build && npm start
